# 🎨 **UIモック（全サイズ対応）
Worldchain × WLD Dashboard**

---

# 🟥 **1. レイアウト前提**

### ✔ カラム構成（カードグリッド）

```
grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
```

→ **カード数がいくら増えても自動で最適配置になる。**

### ✔ 最小幅：360px

→ スマホ最小端末でも完全に収まる

### ✔ 余白

* ページ左右：16px
* カード間：16px
* セクション間：24〜32px

# 🟦 **2. 全端末サイズ別のUIモック（構造表示）**

---

# 📱 **［サイズ：360px（最小スマホ）］**

* 1カラム構成
* カードは縦1枚ずつ
* グラフ類は幅100%
* ヘッダーは1行表示

```
────────────────────
  Worldchain × WLD Dashboard
────────────────────

[ Network Stats ]
┌────────────────┐
│ TPS: 3.21       │
│ 24h TX: 4,520,221│
│ ...              │
└────────────────┘

[ WLD Market ]
┌────────────────┐
│ Price: $2.14     │
│ Change: -1.2%    │
│ ...              │
└────────────────┘

[ Activity Breakdown ]
┌────────────────┐
│ Native: 42%      │
│ Token: 33%       │
│ Contract: 25%    │
└────────────────┘

[ 7d Trend Charts ]
┌────────────────┐
│ WLD Price Chart │ (全幅)
└────────────────┘
┌────────────────┐
│ TX Trend Chart  │
└────────────────┘

[ Alerts ]
┌────────────────┐
│ Spike detected… │
└────────────────┘

Footer
────────────────────
```

**ポイント：**

* 最小画面でもすべて“縦積み”で崩れない
* スパークラインはテキスト左／右に並べず、**カード内下部に横長で配置**

---

---

# 📱 **［サイズ：414〜480px（一般スマホ）］**

* 基本は 1 カラム
* ただし **幅480pxでは一部2カラム化が可能**

```
Network Stats
┌───────┐  ┌───────┐
│ TPS   │  │ 24h TX│
└───────┘  └───────┘

┌───────┐  ┌───────┐
│ Gas   │  │ New Addr│
└───────┘  └───────┘
```

**ポイント：**

* auto-fit により “横に置ける場合のみ” 自動で2カラム化
* 開発側が条件を判定しなくても CSS が勝手にやる

---

---

# 🟧 **［サイズ：600〜768px（タブレット縦）］**

* 2カラム構成へ自然に移行
* カードは2×Nのグリッド
* トレンドチャートは全幅を占有

```
┌───────────────┬───────────────┐
│ TPS             │ 24h TX         │
├───────────────┼───────────────┤
│ New Address     │ Total Address   │
├───────────────┼───────────────┤
│ Gas             │ Price (WLD)     │
└───────────────┴───────────────┘

[ Trend Charts ]（全幅）
┌─────────────────────────────┐
│ WLD Price 7d Chart           │
└─────────────────────────────┘
┌─────────────────────────────┐
│ TX Trend 7d Chart            │
└─────────────────────────────┘
```

**ポイント：**

* 情報密度が上がり「ダッシュボード感」が強まる
* カードは高さ自動伸縮

---

---

# 🟩 **［サイズ：1024px（タブレット横 / 小型PC）］**

* 多くのカードが **3カラム配置** になる
* 情報レイアウトの最適化が自然に発生（CSS任せ）

```
┌──────────┬──────────┬──────────┐
│   TPS      │  24h TX    │  New Addr  │
├──────────┼──────────┼──────────┤
│ Total Addr │ Gas Price  │ WLD Price  │
└──────────┴──────────┴──────────┘

[ Activity Breakdown ]（2カラム）
┌──────────┬──────────┐
│ Native %   │ Token %   │
├──────────┼──────────┤
│ Contract % │ Bridge %  │
└──────────┴──────────┘
```

**ポイント：**

* 3カラムで格段に見やすくなる
* チャートは依然として **横幅100%** で大きく描画

---

# 🟦 **［サイズ：1440px（一般的なPCワイド）］**

* **4カラム構成が自然発生**
* カードは横に広く配置され、「情報密度が高いプロダクト感」が出る

```
┌─────┬─────┬─────┬─────┐

├─────┼─────┼─────┼─────┤
│Gas  │Price│MCap │Vol  │
```

```
┌─────────┬─────────┬─────────┬─────────┐
│Native   │Token    │Contract │Bridge   │
```

→ **横幅最大 / 高さ200–260px** の2段構成

# 🟪 **［サイズ：1920px（大画面 / デスクトップ）］**

* **5カラム表示まで可能**
* ここで“企業ツール並み”の密度になる

```
┌────┬────┬────┬────┬────┐
│TPS │24hTX│New│Total│Gas│
├────┼────┼────┼────┼────┤
│WLD │MCap│Vol │... │...│
└────┴────┴────┴────┴────┘
```

チャートは 2×2 または 1×2構成で横幅を最適化。

* 6カラム構成も可能

```

│TPS │24hTX│New│Total│Gas│Price│
```

* 1段目：横幅 MAX

  など **フレキシブルに組める設計**。

```
</Section>

  - TX Trend Chart
```

# 🔵 **4. レスポンシブ動作まとめ**

| 画面幅       | カラム数 | 説明           |
| --------- | ---- | ------------ |
| 360px     | 1    | 最小構成、縦並び     |
| 414〜480px | 1〜2  | 横に収まる場合は2カラム |
| 600〜768px | 2    | タブレット縦       |
| 1024px    | 3    | 小型PC         |
| 1440px    | 4    | 一般PC         |
| 1920px    | 5    | 大型PC         |
| 2560px〜   | 6    | 超広画面         |

→ **すべて CSS の auto-fit で自動化されるため、コーディングは最小限。**
