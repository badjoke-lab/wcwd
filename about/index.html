About WCWD

WCWD is an independent, unofficial Worldcoin, World Chain, WLD, and World ID toolkit.

It is a static Pages-first site with small read-only API boundaries for explicit user-triggered reads.

Operating model

WCWD is built from static HTML, CSS, JavaScript, and reviewed static data wherever possible. Cloudflare Cron Triggers are zero, there is no Worker scheduled handler, and WCWD does not run background collection on Cloudflare.

Live reads happen only when a user opens a tool or presses a control. Historical or curated snapshots are bounded, manually produced or manually dispatched, reviewed, and labeled with source and freshness semantics.

API and data boundaries

The Pages API boundary is read-only and route allowlisted. Public GET requests must not create persistent writes or run admin work. Unsupported metrics are marked unavailable instead of being filled with synthetic values.

World Chain metrics are best-effort summaries, not a full indexer, official explorer, realtime feed, or financial advice. Token Heatmap remains non-indexable and unavailable unless a reviewed snapshot satisfies its source, timestamp, contract, and freshness requirements.

Privacy

World ID proof JSON is ephemeral to the active page session. WCWD does not persist proof payloads in localStorage, sessionStorage, IndexedDB, Worker storage, analytics, URLs, or query strings.

Analytics, where present, is limited to non-sensitive allowlisted event names. WCWD does not send proof payloads, endpoint URLs, raw responses, or raw errors to analytics.

Support

WCWD is free to use and donation-supported. Optional support helps fund hosting, maintenance, source review, data quality work, and manual bounded snapshot/history work.

Cloudflare Cron and automatic KV history collection are intentionally cancelled architecture paths.
